目的が「仕様駆動開発の入力として使える、a11y/日本語文言が固定されたHTMLモック（できれば状態別）を、生成AIで素早く作る」なら、選定の軸は次の3つです。

1. HTMLをそのまま出せる/編集できる（画像だけにならない）
1. DaisyUI/Tailwindの制約を守らせやすい（スタック固定）
1. a11y（label/role）と日本語文言固定をプロンプト・ルールで担保できる

その前提でおすすめを用途別に挙げます。

# 1) 最も堅い構成（実務で回しやすい）

## A. 「VS Code + GitHub Copilot」＋「Viteの静的HTMLプレビュー」

- **VS Code：** HTMLモックをリポジトリで管理（差分・レビュー・履歴が残る）
- **GitHub Copilot（Chat/Edits）：** DaisyUIのHTML断片生成、表・フォーム・モーダルなどの雛形量産に強い
- **Vite：** /mock を静的に見せる（またはStorybookの代替的に）
  - 1状態モックをすぐブラウザで確認できる

### 良い点

- 生成物が最初から「SpecKit投入可能なテキスト（HTML）」として残る
- 日本語文言やa11y属性を“規約ファイル”として同じリポジトリに置ける
- 実装へ移行するときに流用しやすい（classや文言がそのまま残る）

### 注意点

- “状態別”はファイル複製でよいので、最初から idle/submitting/error/empty を作る運用にすると後工程が安定します

# 2) ノーコード寄りで「モック作成に集中」したい場合

## B. Figma + AI（ただし最終成果物はHTMLに戻す）

- Figma自体は強いですが、AI支援（文言提案、レイアウト案）で“画面の合意形成”には役立ちます
- ただし、あなたの狙いは「SpecKit入力にHTMLを使う」「テストのセレクタを日本語a11y名で固定する」なので、**Figmaだけで完結させると情報が落ちます**
- 使うなら「関係者レビュー・合意形成」用途に限定し、**最終的にHTMLへ転記**するのが安全です

# 3) 生成AIで“UIを直接HTMLで吐かせる”用途に向くサービス

## C. ChatGPT / Claude / Gemini（+ ルール文書を常に添付）

「DaisyUIで、a11y/日本語固定で、状態別HTMLも出せ」と指示して、HTMLを生成→VS Codeへ貼る運用ができます。

### コツ

- 1回で完璧を狙わず、「見出し/フォーム/リスト/エラー表示」などに分割して生成
- あなたの Tech & Rules（日本語固定、label必須、DaisyUI使用）を毎回の入力に含める
- **出力フォーマットを固定**する（例：`<main>`配下に`<h1>`、操作領域、一覧領域、エラー領域、など）

# 4) Tailwind/DaisyUIのモック作成を加速する具体ツール

## DaisyUIのコンポーネントカタログを“参照”にして生成する

- DaisyUIは部品が定型なので、AIに「この部品を使う」と宣言するとブレが減ります<BR>
  例：alert, btn, card, navbar, table, input, loading など

## Tailwind Play / Playground系

- ブラウザ上で Tailwind を当てながら素早く形を作れます
- ただし成果物の管理（差分・レビュー）が弱いので、最終的にはリポジトリへ戻すのがおすすめ

# 5) あなたの目的に最適な“おすすめセット”

結論としてのおすすめはこれです：

1. VS Code + GitHub Copilot（HTMLモックを直接生成・修正）
1. Viteでモックをプレビュー（/mocks を静的ページとして表示）
1. （必要なら）ChatGPT/Claudeを併用して、状態別モックや文言辞書を作る
1. Figmaは合意形成用（最終成果物はHTMLにする）

この構成だと、HTMLモックがそのまま

- UI文言一覧（日本語固定）
- a11y名（label/role）
- 状態別仕様（差分） に落としやすく、前に作ったSpecKitテンプレに直結します。

# 6) すぐ使える「HTMLモック生成プロンプト（雛形）」

普段のAI生成でブレを減らすため、こういう雛形を固定して使うのが効果的です（サービスはCopilot/ChatGPT/ClaudeどれでもOK）。

- 前提：Tailwind + DaisyUI
- 条件：日本語文言固定、label/id必須、a11y優先
- 出力：\<main>直下に h1、操作領域、一覧領域、空状態領域（コメントでOK）

必要なら、このプロンプト雛形をあなたの「タスクリスト」仕様（タスクリスト、再読み込み、タスクがありません 等）に合わせて具体化して渡します。
